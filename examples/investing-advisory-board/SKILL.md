## 模式二：评审会

触发：用户要求多视角；或问题不可逆/金额重大；或路由后用户说"再听听别人的"。

流程：
1. 提炼问题核心，一次性向用户确认关键事实（标的、金额占比、期限、已有判断）
2. 需要外部事实先统一 WebSearch，事实池共享给所有视角
3. spawn 3 个 subagent 各带一个专家 Skill 并行评审（避免视角串味）
4. 输出评审会纪要：

```
## 评审会纪要：[问题]
### 共识
### 分歧（谁 vs 谁，各自依据的心智模型，为什么会分歧）
### 各自核心建议（每人 ≤3 条，保留个人风格）
### 方法论检验（用最相关的书籍清单过一遍结论）
### 综合结论（条件化：如果你更在乎X听A，更在乎Y听B）
### 本次评审的盲区
```

规则：分歧必须呈现张力，禁止和稀泥；综合结论必须条件化，不给"既要又要"。

## 分歧地图（预置）

1. **段永平 vs 马克斯：要不要看周期** — 段：好生意拿着不动，周期是噪音（依据"生意模式第一"）；马克斯：忽视周期位置的买入是不完整的风险评估（依据"钟摆理论"）。落到用户：持有期 10 年+且现金流稳 → 偏段；3-5 年或用到杠杆 → 必须听马克斯。
2. **芒格 vs 格雷厄姆式清单：集中 vs 分散** — 芒格：好机会一生没几次，重仓；格雷厄姆：安全边际只有在组合层面才可靠。落到用户：认知深度够不够重仓，用芒格自己的误判清单先检验。
3. **段永平 vs 芒格：判断方式** — 段用"懂/不懂"二分直觉过滤；芒格要求多模型显式交叉检验。同一标的可能段说"不懂不碰"而芒格给出可分析路径。
