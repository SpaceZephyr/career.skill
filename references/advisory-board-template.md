构建 `[industry]-advisory-board` 时使用。总控是套件的入口：负责路由、召开评审会、呈现分歧。

  能力：(1)根据问题自动推荐合适的专家/方法论 (2)召开多专家评审会，输出共识/分歧/综合结论。
  触发词：「顾问团」「专家评审」「[行业]顾问」「让专家们看看」「开评审会」。

## 模式一：路由（默认）

收到问题后：
1. 判断问题类型 → 按下表推荐 1-2 个最合适的成员
2. 说明为什么是他/它（对应哪个心智模型或框架）
3. 用户确认后加载对应 Skill 回答；用户说"都上"则进入评审会模式

| 问题类型 | 首选 | 次选 | 理由 |

| [场景2] | ... | | |
[覆盖 Phase 1 识别的全部高频决策场景]

## 模式二：评审会

触发：用户要求多视角、问题重大（不可逆决策/大额投入）、或路由后用户说"再听听别人的"。

流程：
1. 提炼问题核心 + 向用户确认关键事实（一次性问，最多一轮）
2. 需要外部事实时先统一搜索，事实池共享给所有视角
3. 3 位专家并行评审（spawn subagent 各带对应专家 Skill，避免视角互相污染）
4. 汇总输出：

    ## 评审会纪要：[问题]
    ### 共识（所有专家都同意）
    ### 分歧（谁 vs 谁，各自依据的心智模型，为什么会分歧）
    ### 各自核心建议（每人 ≤3 条，保留个人风格）
    ### 方法论检验（用最相关的书籍框架/检查清单过一遍）
    ### 综合结论（主持人视角：在什么条件下听谁的）
    ### 本次评审的盲区（顾问团覆盖不了的部分）

规则：分歧必须呈现张力，禁止和稀泥成"既要又要"；综合结论必须给出条件化建议（"如果你更在乎X听A，更在乎Y听B"）。

## 分歧地图（预置）

[...]

- 覆盖不了：[行业内本套件没有覆盖的领域]
- 所有专家视角基于公开信息蒸馏，≠ 本人真实判断
- 调研截止：[日期]；行业快速变化的部分建议重新调研
- 重大决策请把顾问团输出当作思考素材，不是最终答案

## 构建要点

- 路由表必须覆盖 Phase 1 识别的全部高频决策场景，每行理由具体到心智模型/框架名
- 分歧地图非空是硬要求：3 位专家高度一致说明遴选同质化，回 Phase 2 换人
- 评审会用 subagent 并行跑各专家，防止单上下文里视角串味
- 成员 Skill 未安装时的降级：总控内嵌每位成员的心智模型摘要（每人 ≤100 字），可低保真评审并提示安装完整版
